# Admin Guide

- Document owner: Product and Operations
- Last reviewed: 2026-03-24
- Primary use: Administrative guide for dashboard usage, tenant oversight, and incident visibility

## Scope

This guide is for platform or tenant administrators using the admin dashboard for oversight, configuration, and operational review.

## What You Can Do Today

- Sign in to the admin dashboard.
- Review dashboard tiles, live operational views, and alert state.
- Manage routes, vehicles, students, compliance, and video metadata.
- View boards and schools where the role and current implementation allow it.

## Typical Admin Workflow

1. Sign in and review dashboard status.
2. Open routes, vehicles, or students to verify operational setup.
3. Monitor active alerts and live route activity.
4. Review compliance or audit data when required.
5. Drill into specific school or route issues.

## Important Current-State Caveats

- Board and school onboarding flows are incomplete.
- Board-level aggregation is limited.
- Route optimization remains placeholder quality.
- Some cross-tenant behavior relies on gateway enforcement rather than stronger downstream guarantees.

## When to Use the School Operator Guide Instead

If your work is primarily school-level daily dispatch, route oversight, and immediate incident handling, the [school-operator guide](../school-operator/README.md) is the better operational reference.
