## Executive Summary

The SBTM platform is well-aligned with the operational and real-time tracking portions of the STPMS RFP, but has material gaps in the **planning** half of the RFP (GIS, geocoding, boundary management, routing engine, calendar/bell times, scenario test environments) and in the **commercial** half (financial management, invoicing, performance scorecard, rate refresh / CRF). SBTM also adds capabilities the RFP does not require (BLE student presence, panic alerts, video event service) which can be positioned as differentiators.

**Scoreboard**

| Bucket                  | Count          | Notes                                                                                                   |
| ----------------------- | -------------- | ------------------------------------------------------------------------------------------------------- |
| Fully supported         | 25 RFP items   | Core identity, tenancy, real-time tracking, driver/parent surfaces, multilingual, encryption-in-transit |
| Partially supported     | 21 RFP items   | Most planning-side and reporting clauses                                                                |
| Not yet implemented     | 20 RFP items   | Plus 5 of 6 optional modules                                                                            |
| SBTM-only over-delivery | 6 capabilities | BLE, panic, video, geofencing, RLS, event-driven architecture                                           |

## Methodology Notes

1. RFP clause numbers are reproduced verbatim from the source PDF.
2. "Fully supported" requires (a) data model present, (b) running code path, (c) usable surface — even if not yet packaged as a formal RFP-style report.
3. Items where data exists but a customer-visible report/template is missing are placed in Section 2 (data-availability fully supported) with a forward reference to Wave 1 packaging work.
4. Optional STPMS modules (RFP §2.2.9.1 – 2.2.9.6) are treated as "Not yet implemented" except 2.2.9.5 which is already over-delivered.
5. Effort, sequencing, and acceptance criteria for closing Sections 3 and 4 live in the companion document [STPMS-RFP-ImplementationPlan.md](STPMS-RFP-ImplementationPlan.md).
